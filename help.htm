Описание программы

Программа Blind_Log предназначена для ведения журнала радиосвязей (QSO). Она позволяет добавлять, редактировать, удалять и экспортировать записи QSO в формате ADIF.

Интерфейс программы

Когда программа открывается, фокус попадает на вкладку "Добавить QSO". В этой вкладке можно ввести данные о радиосвязи, такие как позывной, имя, город, QTH, частота, RST принято, RST передано, режим, диапазон и комментарий.

Переходя по вкладкам с помощью клавиш Ctrl+Tab и Ctrl+Shift+Tab, фокус перемещается между вкладками "Добавить QSO" и "Журнал".

Вкладка "Добавить QSO":

  • Фокус по умолчанию находится на поле "Позывной".
  • Используя клавишу Tab, можно перемещаться между полями ввода: "Позывной", "Имя", "Город", "QTH", "Частота", "RST принято", "RST передано", "режим", "диапазон", "Комментарий", "Дата", "Время" и кнопка "добавить QSO".
  • Клавиша Shift+Tab позволяет перемещаться в обратном порядке.

Вкладка "Журнал":

  • Фокус по умолчанию находится на списке записей QSO.
  • Используя клавишу Tab, можно перемещаться между этим списком и кнопками "Редактировать" и "Удалить".
  • Клавиша Shift+Tab позволяет перемещаться в обратном порядке.

Для экспорта журнала используйте горячую клавишу Ctrl+S на вкладке "Журнал".

Настройки интерфейса

Откройте настройки (Ctrl+P). Диалог настроек содержит две вкладки, переключаемые клавишами Ctrl+Tab и Ctrl+Shift+Tab.

Вкладка "Общие":

  • Позывной оператора, имя, QTH, город, оборудование (для информации об окне QSO).
  • Координаты (широта, долгота) для экспорта в ADIF.
  • Настройки часового пояса (UTC или собственный диапазон часов).
  • Опция поиска позывных в QRZ.ru с логином и паролем.
  • Опции логирования и проверки обновлений при запуске.

Вкладка "Интерфейс":

  • Флажки видимости полей: позывной (обязательный), имя, город, QTH, частота, диапазон, режим, RST-принято, RST-передано, комментарий, дата, время.
  • По умолчанию все поля видимы. Снимите флажок, чтобы скрыть поле в форме добавления QSO и в колонках журнала.
  • Примечание: поле "Позывной" всегда видимо и обязательно. Поэтому флажка для этого поля в настройках интерфейса нет.
  • При скрытии полей "Дата" и "Время" система автоматически использует текущие дату и время (с учётом часового пояса из настроек) при добавлении QSO и экспорте в ADIF.

После изменения видимости полей нажмите "Сохранить" — интерфейс перестроится немедленно.

Обязательные поля

При добавлении QSO обязательно необходимо заполнить поле "Позывной". Остальные поля заполняются опционально. Если в интерфейсе скрыты какие-либо опциональные поля, они не будут показаны в форме, но могут быть заполнены программой автоматически (например, дата и время, если они скрыты).

Транслитерация русского текста

Программа поддерживает автоматическую транслитерацию русского текста в латиницу. Это нужно для совместимости с международными системами логирования (LoTW, Club Log и другие), которые используют латинский алфавит.

Как это работает:

  • При добавлении QSO русский текст в полях Имя, Город, и Комментарий автоматически преобразуется в латиницу.
  • Например: "дмитрий" → "dmitrij", "москва" → "moskva".
  • Поле Позывной остаётся без изменений (обычно уже содержит буквы и цифры).
  • Транслитерация происходит только если текст содержит кириллицу. Английский текст остаётся без изменений.
  • При экспорте в ADIF все эти поля содержат уже транслитерированный текст.

Эта функция включена по умолчанию и не требует дополнительных настроек. Она срабатывает автоматически при вводе русского текста.

Сохранение журнала при выходе

Если в журнале есть хотя бы одна запись, при попытке выхода программа предложит сохранить журнал в формате ADIF, выйти без сохранения или отменить выход.

  • Сохранить — откроется диалог сохранения ADIF, после успешного экспорта программа закроется.
  • Не сохранять — программа закроется без сохранения журнала.
  • Отмена — выход будет отменён, вы сможете продолжить работу.

Автосохранение сеанса (temp)

Программа поддерживает функцию автоматического сохранения текущей сессии работы. Это помогает восстановить данные, если программа вылетит или компьютер перезагрузится.

Как включить:

  • Откройте настройки (Ctrl+P).
  • На вкладке "Общие" установите флажок «Автосохранение сеанса (temp)».
  • Нажмите "Сохранить".

Как это работает:

  • Когда автосохранение включено, программа автоматически копирует все записи QSO в скрытый файл blind_log_temp.json рядом с исполняемым файлом.
  • Это происходит каждый раз, когда вы добавляете или удаляете QSO.
  • Файл обновляется в реальном времени, поэтому даже при аварийном выходе теряются только данные, которые не успели добавиться.

Восстановление после сбоя:

  • При следующем запуске программы, если авто-temp включено и файл temp содержит данные, вам будет предложено восстановить сеанс.
  • Диалог покажет количество сохранённых QSO: "Найдены несохранённые данные (N QSO). Восстановить?".
  • Нажмите "Да" для восстановления или "Нет" для начала с нуля.

Очистка temp:

  • После успешного экспорта в ADIF файл temp автоматически удаляется (чтобы не предлагать восстановление пустых данных).
  • Если вы восстановили данные, temp-файл также будет очищен, чтобы не спрашивать о восстановлении дважды.

Важно: Автосохранение — это защита от сбоев, а не замена регулярного экспорта. Рекомендуется экспортировать журнал в ADIF после каждой сессии логирования, чтобы иметь надёжный backup.

Горячие клавиши

  • Ctrl+P - Открыть настройки
  • Ctrl+Q - Выйти из программы
  • Ctrl+Enter - Добавить QSO
  • Ctrl+E - Редактировать выбранное QSO
  • Ctrl+S - Экспортировать QSO в ADIF (только через горячую клавишу)
  • Delete - Удалить выбранное QSO
  • Shift+F1 - О программе
  • F1 - Справка
  • CTRL+F1 - показывает changelog - журнал изменений в программе
  • Ctrl+Tab - Переключение между вкладками вперёд (главное окно и окно настроек)
  • Ctrl+Shift+Tab - Переключение между вкладками назад (главное окно и окно настроек)
  • Ctrl+U - Проверить обновления
  • Enter - Подставить имя и город по позывному из QRZ.ru (если включена настройка)
